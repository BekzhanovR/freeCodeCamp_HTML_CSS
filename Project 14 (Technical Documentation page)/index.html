Kirish

JavaScript oʻzaro platformali, obyektga yoʻnaltirilgan skript tilidir. Bu kichik va engil til. Xost muhitida (masalan, veb-brauzer) JavaScript-ni o'z muhiti ob'ektlari ustidan dasturiy boshqaruvni ta'minlash uchun ulash mumkin.

JavaScript-da Array, Date va Math kabi ob'ektlarning standart kutubxonasi va operatorlar, boshqaruv tuzilmalari va bayonotlar kabi til elementlarining asosiy to'plami mavjud. Asosiy JavaScript-ni qo'shimcha ob'ektlar bilan to'ldirish orqali turli maqsadlar uchun kengaytirilishi mumkin; masalan:

  • Mijoz tomoni JavaScript brauzerni va uning Hujjat Ob'ekt Modelini (DOM) boshqarish uchun ob'ektlarni taqdim etish orqali asosiy tilni kengaytiradi. Masalan, mijoz tomoni kengaytmalari dasturga elementlarni HTML formasiga joylashtirish va sichqonchani bosish, shakl kiritish va sahifa navigatsiyasi kabi foydalanuvchi hodisalariga javob berishga imkon beradi.

  • Server tomonidagi JavaScript serverda JavaScript-ni ishga tushirish uchun tegishli ob'ektlarni taqdim etish orqali asosiy tilni kengaytiradi. Masalan, server tomoni kengaytmalari ilovaga ma'lumotlar bazasi bilan aloqa qilish imkonini beradi, dasturning bir chaqiruvidan ikkinchisiga ma'lumotlarning uzluksizligini ta'minlaydi yoki serverda fayllar bilan ishlashni amalga oshiradi.
Siz allaqachon bilishingiz kerak bo'lgan narsa

Ushbu qo'llanma sizda quyidagi asosiy ma'lumotlarga ega ekanligingizni taxmin qiladi:

  • Internet va World Wide Web (WWW) haqida umumiy tushuncha.

  • HyperText Markup Language (HTML) bo‘yicha yaxshi ish bilimi.

  • Ba'zi dasturlash tajribasi. Agar siz dasturlashda yangi bo'lsangiz, JavaScript haqida asosiy sahifada havola qilingan o'quv qo'llanmalaridan birini sinab ko'ring.
JavaScript va Java

JavaScript va Java ba'zi jihatdan o'xshash, ammo ba'zilarida tubdan farq qiladi. JavaScript tili Java-ga o'xshaydi, lekin Java-ning statik yozish va kuchli turdagi tekshiruvi mavjud emas. JavaScript ko'pgina Java ifodasi sintaksisiga, nomlash qoidalariga va asosiy boshqaruv oqimi konstruksiyalariga amal qiladi, bu uning LiveScript-dan JavaScript-ga qayta nomlanishiga sabab bo'lgan.

Java-ning deklaratsiyalar bo'yicha tuzilgan kompilyatsiya vaqtidagi sinflar tizimidan farqli o'laroq, JavaScript raqamli, mantiqiy va satr qiymatlarini ifodalovchi oz sonli ma'lumotlar turlariga asoslangan ish vaqti tizimini qo'llab-quvvatlaydi. JavaScript-da keng tarqalgan sinfga asoslangan ob'ekt modeli o'rniga prototipga asoslangan ob'ekt modeli mavjud. Prototipga asoslangan model dinamik merosni ta'minlaydi; ya'ni meros bo'lgan narsa alohida ob'ektlar uchun farq qilishi mumkin. JavaScript, shuningdek, hech qanday maxsus deklarativ talablarsiz funksiyalarni qo'llab-quvvatlaydi. Funktsiyalar erkin terilgan usullar sifatida bajariladigan ob'ektlarning xossalari bo'lishi mumkin.

JavaScript Java bilan solishtirganda juda erkin shaklli tildir. Barcha o'zgaruvchilar, sinflar va usullarni e'lon qilishingiz shart emas. Usullarning umumiy, shaxsiy yoki himoyalanganligi haqida tashvishlanishingiz shart emas va interfeyslarni amalga oshirishingiz shart emas. O'zgaruvchilar, parametrlar va funktsiyani qaytarish turlari aniq yozilmagan.

Salom Dunyo

JavaScript yozishni boshlash uchun Scratchpad-ni oching va birinchi "Salom dunyo" JavaScript kodingizni yozing:

function greetMe(yourName) { alert("Hello " + yourName); }
greetMe("World");

Kodni brauzeringizda ko'rish uchun paneldagi kodni tanlang va Ctrl+R tugmalarini bosing!

O'zgaruvchilar

Siz o'zgaruvchilardan ilovangizdagi qiymatlar uchun ramziy nom sifatida foydalanasiz. Identifikatorlar deb ataladigan o'zgaruvchilarning nomlari ma'lum qoidalarga mos keladi.

JavaScript identifikatori harf, pastki chiziq (_) yoki dollar belgisi ($) bilan boshlanishi kerak; keyingi belgilar ham raqamlar bo'lishi mumkin (0-9). JavaScript katta-kichik harflarga sezgir bo'lganligi sababli, harflar "A" dan "Z"gacha (katta harf) va "a" dan "z"gacha (kichik harf) belgilarni o'z ichiga oladi.

Identifikatorlarda ISO 8859-1 yoki å va ü kabi Unicode harflaridan foydalanishingiz mumkin. Unicode qochish ketma-ketliklarini identifikatorlarda belgilar sifatida ham ishlatishingiz mumkin. Yuridik nomlarning ba'zi misollari: Number_hits, temp99 va _name.

O'zgaruvchilarni e'lon qilish

O'zgaruvchini uchta usulda e'lon qilishingiz mumkin:

var kalit so'zi bilan. Masalan,

var x = 42.

Ushbu sintaksis mahalliy va global o'zgaruvchilarni e'lon qilish uchun ishlatilishi mumkin.

Unga oddiygina qiymat berish orqali. Masalan,

x = 42.

Bu har doim global o'zgaruvchini e'lon qiladi. Bu qattiq JavaScript ogohlantirishini yaratadi. Siz bu variantni ishlatmasligingiz kerak.

Let kalit so'zi bilan. Masalan,

let y = 13.

Ushbu sintaksis blok doirasi mahalliy o'zgaruvchisini e'lon qilish uchun ishlatilishi mumkin. Quyidagi o'zgaruvchan doiraga qarang.

O'zgaruvchan doira

Har qanday funktsiyadan tashqarida o'zgaruvchini e'lon qilganingizda, u global o'zgaruvchi deb ataladi, chunki u joriy hujjatdagi istalgan boshqa kod uchun mavjud. Funktsiya ichida o'zgaruvchini e'lon qilganingizda, u mahalliy o'zgaruvchi deb ataladi, chunki u faqat shu funktsiya ichida mavjud.

ECMAScript 2015 dan oldingi JavaScript blok bayonot doirasiga ega emas; aksincha, blok ichida e'lon qilingan o'zgaruvchi blokda joylashgan funktsiya (yoki global qamrov) uchun lokaldir. Misol uchun, quyidagi kod log 5 bo'ladi, chunki x doirasi blok emas, balki x e'lon qilingan funksiya (yoki global kontekst) bo'lib, bu holda if ko'rsatmasi hisoblanadi.

if (true) { var x = 5; } console.log(x); // 5

ECMAScript 2015 da kiritilgan let deklaratsiyasidan foydalanganda bu xatti-harakat o'zgaradi.

if (true) { let y = 5; } console.log(y);
// ReferenceError: y is not defined
Global o'zgaruvchilar

Global o'zgaruvchilar aslida global ob'ektning xususiyatlari. Veb-sahifalarda global obyekt oynadir, shuning uchun siz window.variable sintaksisi yordamida global o'zgaruvchilarni o'rnatishingiz va ularga kirishingiz mumkin.

Shunday qilib, bir oyna yoki ramkada e'lon qilingan global o'zgaruvchilarga oyna yoki ramka nomini ko'rsatish orqali boshqa oyna yoki ramkadan kirishingiz mumkin. Misol uchun, agar hujjatda phoneNumber deb nomlangan o'zgaruvchi e'lon qilingan bo'lsa, iframe'dan ushbu o'zgaruvchiga parent.phoneNumber sifatida murojaat qilishingiz mumkin.

Konstantalar

Const kalit so'zi bilan faqat o'qish uchun mo'ljallangan konstantani yaratishingiz mumkin. Doimiy identifikatorning sintaksisi o'zgaruvchi identifikatori bilan bir xil: u harf, pastki chiziq yoki dollar belgisi bilan boshlanishi kerak va alfavit, raqamli yoki pastki chiziq belgilaridan iborat bo'lishi mumkin.

const PI = 3.14

Skript ishlayotgan vaqtda doimiy qiymatni belgilash orqali o'zgartira olmaydi yoki qayta e'lon qilinishi mumkin emas. U qiymatga ishga tushirilishi kerak.

Konstantalar uchun qamrov qoidalari ruxsat bloki doirasi o'zgaruvchilari bilan bir xil. Agar const kalit so'zi o'tkazib yuborilsa, identifikator o'zgaruvchini ifodalaydi deb taxmin qilinadi.

Siz bir xil nomdagi funktsiya yoki o'zgaruvchi bilan bir xil doiradagi doimiyni e'lon qila olmaysiz. Masalan:

// THIS WILL CAUSE AN ERROR function f() {}; const f = 5;
// THIS WILL CAUSE AN ERROR ALSO function f() { const g = 5; var g;
//statements }

Biroq, ob'ekt atributlari himoyalanmagan, shuning uchun quyidagi bayonot muammosiz bajariladi.

const MY_OBJECT = {"key": "value"};
MY_OBJECT.key = "otherValue";
Ma'lumotlar turlari

Eng so'nggi ECMAScript standarti ettita ma'lumot turini belgilaydi:

  • Primitiv bo'lgan oltita ma'lumot turi:

    • Boolean. rost va yolg'on.

    • null. Null qiymatni bildiruvchi maxsus kalit so'z. JavaScript katta-kichik harflarni sezgir bo'lgani uchun null Null, NULL yoki boshqa variantlar bilan bir xil emas.

    • undefined. Qiymati aniqlanmagan yuqori darajadagi xususiyat.

    • Number. 42 yoki 3.14159.

    • String. "Salom"

    • Symbol (ECMAScript 2015 da yangi). Namunalari noyob va o'zgarmas bo'lgan ma'lumotlar turi.

  • va Obeyktlar

Ushbu ma'lumotlar turlari nisbatan kichik bo'lsa-da, ular ilovalaringiz bilan foydali funktsiyalarni bajarishga imkon beradi. Ob'ektlar va funktsiyalar tilning boshqa asosiy elementlari hisoblanadi. Ob'ektlarni qiymatlar uchun nomlangan konteynerlar va funktsiyalarni ilovangiz bajaradigan protseduralar sifatida ko'rishingiz mumkin.

If-else bayonoti

Agar mantiqiy shart rost bo'lsa, ifodani bajarish uchun if operatoridan foydalaning. Agar shart noto'g'ri bo'lsa, bayonotni bajarish uchun ixtiyoriy else bandidan foydalaning. If iborasi quyidagicha ko'rinadi:

if (condition) { statement_1; } else { statement_2; }

shart rost yoki noto'g'ri deb baholanadigan har qanday ifoda bo'lishi mumkin. To'g'ri va noto'g'ri deb baholanadigan narsalarni tushuntirish uchun Booleanga qarang. Agar shart rost deb baholansa, operator_1 bajariladi; aks holda, operator_2 bajariladi. bayonot_1 va bayonot_2 har qanday bayonot bo'lishi mumkin, shu jumladan keyingi o'rnatilgan if iboralari.

Bundan tashqari, bir nechta shartlarni ketma-ket sinovdan o'tkazish uchun else if dan foydalanib, quyidagi tarzda ifodalarni birlashtirishingiz mumkin:

if (condition_1) { statement_1; } else if (condition_2) { statement_2; } else if (condition_n) { statement_n; } else { statement_last; }
While bayonoti

While bayoni belgilangan shart rost deb baholanguncha o'z bayonotlarini bajaradi. while bayonoti quyidagicha ko'rinadi:

while (condition) statement

Agar shart noto'g'ri bo'lsa, sikl ichidagi operator bajarilishini to'xtatadi va boshqaruv tsikldan keyingi operatorga o'tadi.

Shart sinovi tsikldagi bayonot bajarilishidan oldin sodir bo'ladi. Agar shart rost bo'lsa, bayonot bajariladi va shart yana sinovdan o'tkaziladi. Agar shart noto'g'ri bo'lsa, bajarish to'xtaydi va boshqaruv bir vaqtning o'zida keyingi bayonotga o'tkaziladi.

Bir nechta iboralarni bajarish uchun ushbu bayonotlarni guruhlash uchun blok bayonotidan ({ ... }) foydalaning.

Misol:

Quyidagi while tsikli n uchtadan kichik bo'lganda takrorlanadi:

var n = 0; var x = 0; while (n < 3) { n++; x += n; }

Har bir iteratsiya bilan tsikl n ni oshiradi va bu qiymatni x ga qo'shadi. Shunday qilib, x va n quyidagi qiymatlarni qabul qiladi:

  • Birinchi o'tishdan keyin: n = 1 va x = 1
  • Ikkinchi o'tishdan keyin: n = 2 va x = 3
  • Uchinchi o'tishdan keyin: n = 3 va x = 6

Uchinchi o'tishni tugatgandan so'ng, n < 3 sharti endi to'g'ri emas, shuning uchun tsikl tugaydi.

Funktsiya deklaratsiyasi

Funktsiya ta'rifi (funktsiya deklaratsiyasi yoki funktsiya bayonoti deb ham ataladi) funktsiya kalit so'zidan iborat bo'lib, undan keyin:

  • Funktsiya nomi.
  • Qavslar ichiga olingan va vergul bilan ajratilgan funktsiyaga argumentlar ro'yxati.
  • Funktsiyani belgilaydigan JavaScript iboralari jingalak qavslar ichida { }.

Masalan, quyidagi kod kvadrat nomli oddiy funktsiyani belgilaydi:

function square(number) { return number * number; }

Funktsiya kvadrati raqam deb ataladigan bitta argumentni oladi. Funktsiya argumentini (ya'ni raqamni) o'ziga ko'paytirilishini aytadigan bitta bayonotdan iborat. Qaytish bayonoti funksiya tomonidan qaytarilgan qiymatni belgilaydi.

return number * number;

Funktsiyalarga ibtidoiy parametrlar (masalan, raqam) qiymat bo'yicha uzatiladi; qiymat funksiyaga uzatiladi, lekin funksiya parametr qiymatini o‘zgartirsa, bu o‘zgarish global miqyosda yoki chaqiruvchi funksiyada aks ettirilmaydi.
